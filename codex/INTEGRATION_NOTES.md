# Интеграционные заметки

## Цель пакета

Встроить новые задачи UI/loads/console после задачи 9.2 без потери уже существующих невыполненных задач 10.1–12.2.

## Что изменено относительно предыдущего архива

1. Добавлен `codex/IMPLEMENTATION_PLAN.md` с полным порядком задач.
2. Новые задачи занимают диапазон 10.1–17.2.
3. Старые задачи 10.1–12.2 перенесены в диапазон 18.1–20.2.
4. Старый `12-01-model-docs.md` обновлен под `GridEngModel v0.2`, потому что после новой задачи 10.2 документация v0.1 становится устаревшей.
5. Старый `11-02-stress-map-mode.md` теперь должен использовать единый MUI Alert/Snackbar для предупреждений.

## Рекомендуемый способ применения

Скопировать папку `codex/` из архива поверх существующей папки `codex/` проекта. После этого текущие файлы `codex/prompts/10-*.md`, `11-*.md`, `12-*.md` будут заменены новыми задачами, а старые задачи будут доступны как `18-*.md`, `19-*.md`, `20-*.md`.

## Контроль

Перед запуском Codex можно проверить список задач:

```bash
ls codex/prompts | sort
sed -n '1,220p' codex/IMPLEMENTATION_PLAN.md
```
