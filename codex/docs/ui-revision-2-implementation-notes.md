# UI revision 2 — implementation notes

Этот документ фиксирует итоговое состояние задач `21.1–21.8`.

## TopMenu

- Лента собрана из групп `Компонент`, `Режим просмотра`, `Видимость`, `Каталог`, `Аналитика`, `Отчет`, `О программе`.
- Большие кнопки имеют единый размер, вторичные команды живут либо в правой колонке, либо в dropdown-меню группы.
- Placeholder-команды используют единый `MUI Alert/Snackbar` info-style.

## Active Load Case

- `activeLoadCaseId` живет в `modelStore`.
- ProjectTree, PropertiesPanel, wind editor, viewport loads и stress-map читают именно активное загружение, а не `loadCases[0]`.
- Переключение активного загружения не меняет онтологию модели и не дублирует load-state по разным виджетам.

## Visibility Labels

- `visibility.labels` управляет пользовательскими подписями на сцене:
  - силами;
  - моментами;
  - распределенными нагрузками;
  - подписями закреплений.
- Подписи осей не зависят от `visibility.labels`; ими управляет `visibility.axes`.

## Scene Overlays

- `stress legend` закреплена слева снизу.
- `gizmo` закреплен справа снизу.
- `wind overlay` отображает направление и давление ветра активного загружения в HUD-пространстве viewport.

## Console Model

- Docked-консоль встроена в центральную область `CadShell` и по умолчанию закрыта.
- Кнопка `Консоль` в ribbon только toggl'ит docked-консоль.
- Fullscreen-консоль открывается только из кнопки expand внутри консоли.
- Обе консоли используют единый `ConsoleView`, общую историю, лог и command registry.

## Placeholder Commands

Единый info-placeholder подтвержден для команд:

- `Открыть`
- `Сохранить`
- `Экспорт DXF`
- `Каталог конструкций`
- `Сводка`
- `Отчет`
- `Спецификация`
- `Справка`
- `О программе`
- `Лицензия`

Команды `Каталог профилей` и `Каталог материалов` открывают реальные project dialogs и не считаются placeholder.
