# UI revision 3.1 — implementation notes

Этот документ фиксирует итоговое состояние задач `23.1–23.5`.

## DXF import dialog

- Структура диалога: шапка -> tabs `Модель | Профили | Логи` -> нижний блок icon-button действий.
- Служебная информация и диагностические детали не дублируются в `Модели` и `Профилях`; они собраны во вкладке `Логи`.

## Preview normalization and rotation

- DXF preview отображается как временная сцена и не изменяет `modelStore` до нажатия `Импорт`.
- После построения preview применяется нормализация с правилом `Z >= 0`.
- Смещение preview пересчитывается после каждого поворота, поэтому модель остается поднятой над плоскостью `XY`.
- Повороты доступны вокруг осей `X`, `Y`, `Z`; есть отдельный reset.

## Profiles tab

- Вкладка `Профили` оформлена как таблица `Цвет | Тип профиля | Стандарт | Профиль`.
- Колонка `Цвет` содержит только swatch.
- Клик по swatch открывает color picker и меняет только preview-окраску группы, не исходные DXF-данные.
- `Тип профиля` и `Стандарт` работают как фильтры каталога.
- Если выбранный профиль больше не соответствует фильтрам, назначение сбрасывается.
- При текущем локальном каталоге `Стандарт` отображается как read-only `Локальный каталог`.

## Logs tab

- Вкладка `Логи` показывает:
  - summary импорта;
  - параметры DXF import;
  - preview rotation и normalization shift;
  - группы `цвет -> профиль`;
  - компактные агрегированные warnings/errors по import, `LINE`, элементам и узлам.
- Raw JSON, dumps store и debug-структуры в пользовательском UI не выводятся.
- Для color/profile mapping в логах сохраняется исходный DXF color, даже если пользователь меняет preview swatch в таблице `Профили`.

## Wind dialog

- Диалог называется `Ветер`.
- Над select загружений нет отдельной подписи.
- В dropdown показываются только `loadCase.name`, внутренним value остается `loadCase.id`.
- Поле `Z` в UI отсутствует; при сохранении всегда уходит `direction.z = 0`.
- Предупреждение о ненулевом `Z` сохраняется только для случаев миграции/старых данных.
- Информационный alert из формы удален.
- Давление вводится и хранится в `Па`, `0` допустим, отрицательные значения запрещены, шаг `10 Па`.
- Кнопки действий остаются icon-button с tooltip и `aria-label`.

## Manual checklist

1. Открыть DXF import dialog и проверить tabs `Модель | Профили | Логи`.
2. Убедиться, что во вкладке `Модель` 3D preview занимает основное пространство.
3. Загрузить DXF и проверить нормализацию: `Z >= 0`, preview не уходит ниже плоскости `XY`.
4. Проверить повороты вокруг `X/Y/Z` и reset.
5. Проверить, что во вкладке `Профили` есть только колонки `Цвет | Тип профиля | Стандарт | Профиль`.
6. Нажать на swatch во вкладке `Профили` и убедиться, что цвет меняется в таблице и в режиме preview `Профили`.
7. Убедиться, что счетчики, layer/meta и служебные summary не показываются во вкладке `Профили`.
8. Открыть `Логи` и проверить compact sections без raw JSON.
9. Проверить, что сообщения по одному элементу агрегированы в одну карточку.
10. Проверить, что в логах для color/profile mapping виден исходный DXF color.
11. Нажать нижнюю кнопку `Логи` и убедиться, что активируется вкладка `Логи`.
12. Открыть диалог `Ветер` и проверить отсутствие подписи над dropdown.
13. Проверить, что dropdown `Ветер` показывает только имена загружений.
14. Проверить, что информационный alert в форме `Ветер` отсутствует, а warning про старый `Z` появляется только при ненулевом `direction.z`.
15. Проверить, что давление допускает `0`, отклоняет отрицательные значения и меняется шагом `10 Па`.
16. Выполнить `cd frontend && npm run build`.
17. Выполнить `cd frontend && npm run lint`.
