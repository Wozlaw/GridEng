# Task 22.6 — Wind dialog polish and UI revision 3 docs

## Цель

Доработать диалог ввода параметров ветра и закрыть ревизию 3 документацией/checklist без изменения 3D-модели ветрового вектора.

## Диалог ветра

1. Заголовок заменить на `Ветер`.
2. Поле `Загружение` отодвинуть от заголовка; все вертикальные отступы формы выровнять через единый `Stack spacing`, без точечных margin-исключений.
3. Поле `Z` убрать из UI.
4. При сохранении всегда передавать `direction.z = 0`.
5. Тип модели/API ветра не менять: 3D-вектор направления сохраняется в модели, ограничение `Z=0` — только UI-правило текущей итерации.
6. Для существующих моделей с ненулевым `z` при открытии диалога показать предупреждение в форме или логах, а при сохранении явно нормализовать к `0`.
7. Форму сделать уже примерно на треть, сохранив адаптивность для узких экранов (`xs: 100%`).
8. Запретить отрицательное значение давления:
   - input-level validation `min=0`;
   - submit-level validation;
   - при ручном вводе отрицательного числа показывать validation error, а не молча игнорировать.
9. `0` Па допускается как давление отсутствует/ветер отключен.
10. Поля `X`, `Y`, `номинальное давление` сделать через `NumberField from './components/NumberField'` или существующий проектный/MUI wrapper с тем же назначением.
11. Инкременты:
    - `X`: `step=1`;
    - `Y`: `step=1`;
    - номинальное давление: `step=10`.
12. Единицы давления — Па. Поле модели должно быть `nominalPressurePa`; все UI, валидация и документация должны использовать `Па` без обратной конверсии в `кПа`.
13. Кнопки сделать icon button с `Tooltip` и `aria-label`.
14. В выпадающем списке загружений показывать только наименования загружений.
15. Внутренний `value` select должен оставаться `loadCase.id`.
16. Если имена загружений дублируются, id можно показывать только в tooltip/debug/logs, не в основном тексте списка.
