# Task 15.2 — Визуальный язык нагрузок и закреплений

## Цель

Обновить уже реализованную визуализацию из задач 9.1/9.2 под новую модель нагрузок и требования по закреплениям.

## Нагрузки

1. Все нагрузки отображать в ГСК.
2. Узловая сосредоточенная force-нагрузка:
   - стрелка в direction;
   - длина/масштаб по magnitude;
   - цветовой акцент force.
3. Узловая moment-нагрузка:
   - дуговая стрелка/моментный маркер;
   - направление по вектору момента;
   - цветовой акцент moment.
4. Member distributed linear:
   - отображать вдоль выбранного участка стержня `xStartRel..xEndRel`;
   - показывать начальную и конечную величины `qStart/qEnd`;
   - равномерная нагрузка — одинаковые стрелки;
   - линейная — стрелки/плашка меняются от qStart к qEnd;
   - target member подсвечивается при выборе нагрузки.
5. Для `function` distribution показывать безопасный placeholder/не визуализировать с warning в консоль/alert, если такая нагрузка попала в модель.
6. При выключенной видимости нагрузок скрывать все load markers.

## Закрепления

1. Закрепления хранятся в `restraints[]`, связь по `restraint.nodeId`.
2. Отображать 6 степеней свободы:
   - UX/UY/UZ — поступательные;
   - RX/RY/RZ — вращательные.
3. Визуальный язык:
   - отдельные маркеры/штрихи/кольца по осям;
   - tooltip или label с `UX UY UZ RX RY RZ`;
   - не один общий значок «заделка».
4. При выборе node с restraint подсвечивать restraint.
5. При выборе restraint подсвечивать node.
6. При выключенной видимости закреплений скрывать restraint markers.

## Проверка

- В sample видны узловые force/moment и распределенная нагрузка.
- Визуализация не меняет расчетную модель.
- `npm run build` и `npm run lint` проходят.
