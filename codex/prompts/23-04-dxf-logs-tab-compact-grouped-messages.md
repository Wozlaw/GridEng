# Task 23.4 — DXF logs tab compact grouped messages

## Цель

Сделать вкладку `Логи` компактной и полезной: сгруппировать сообщения, убрать JSON и лишнюю техническую детализацию из пользовательского вида.

## Требования

1. Вкладка `Логи` должна заменить отдельную перегруженную область предупреждений/служебной информации в основном окне.
2. Логи сделать компактными и визуально аккуратными:
   - плотный список/accordion/table sections;
   - лаконичные заголовки групп;
   - grayscale UI, но сохранить диагностические статусы warning/error.
3. Сообщения по одному элементу объединять:
   - один member/node/color group → одна строка/карточка с агрегированным описанием;
   - не выводить несколько однотипных сообщений подряд для одного элемента.
4. Не выводить raw JSON, dumps объектов, внутренние store snapshots и длинные технические структуры.
5. Содержимое логов:
   - summary импорта;
   - параметры DXF settings;
   - параметры preview normalization/rotation;
   - warnings/errors;
   - ignored entities count/types;
   - color/profile mapping diagnostics;
   - сведения о нулевых/ошибочных линиях, висячих стержнях, неназначенных профилях.
6. Для каждого сообщения по возможности показывать человекочитаемый контекст:
   - тип элемента;
   - DXF handle/layer/color, если доступен;
   - краткое описание проблемы;
   - статус `info`/`warning`/`error`.
7. Если для логов уже сделан `ImportLogDialog`/`ImportLogDrawer` в задаче 22.2, переиспользовать его содержимое во вкладке `Логи` или вынести общий компонент `ImportLogView`.
8. Кнопка `Логи` из нижнего блока может переключать активную вкладку `Логи`, а не открывать отдельное окно, если tabs уже присутствуют в диалоге.

## Ограничения

- Не превращать вкладку `Логи` в developer console.
- Не добавлять raw JSON toggle в пользовательский UI; debug/raw можно оставить только за существующим debug-флагом, если он уже есть.
- Не дублировать одну и ту же служебную информацию на вкладках `Модель`/`Профили`.

## Критерии готовности

- Логи читаются компактно и без JSON.
- Сообщения одного элемента агрегируются.
- Summary, settings, normalization, warnings/errors и profile diagnostics доступны во вкладке `Логи`.
- Нижняя кнопка `Логи` открывает/активирует вкладку логов.
- Build/lint проходят.
