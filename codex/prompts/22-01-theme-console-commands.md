# Task 22.1 — Theme console commands

## Цель

Добавить консольное управление темой оформления без изменения текущей онтологии UI и без возврата к цветной теме.

## Требования

1. Добавить команды через существующий `command registry`:
   - `theme light` — включить светлый grayscale-режим;
   - `theme dark` — включить темный grayscale-режим;
   - `theme toggle` — переключить режим;
   - `theme status` — показать текущий режим.
2. Команды должны быть доступны из `CommandConsole` и отражаться в `help`/справке консоли.
3. Состояние темы хранить централизованно в существующем app/settings store или ближайшем уже принятом месте хранения UI-настроек.
4. `ThemeProvider` должен читать режим реактивно; не делать локальные `useState`-переключатели темы внутри отдельных компонентов.
5. Сохранить black & white / grayscale эстетику для обоих режимов:
   - `light` — светлый фон, черно-серая типографика и разделители;
   - `dark` — темный фон, серо-белая типографика и разделители;
   - не добавлять цветные primary/accent-палитры.
6. Для `theme status` вывести короткий Alert/Snackbar или console output в существующем стиле.

## Ограничения

- Не менять структуру `TopMenu` и `RIBBON_COMMAND_GROUPS`, если это не требуется для регистрации команды.
- Не дублировать команды отдельными UI-кнопками.
- Не менять бизнес-логику модели/сцены.

## Критерии готовности

- `theme light`, `theme dark`, `theme toggle`, `theme status` работают через консоль.
- `help` показывает новые команды.
- Переключение темы не ломает существующие панели, сцену, docked/fullscreen console.
- Build/lint проходят.

## Ручная проверка

1. Открыть приложение.
2. Ввести `theme status`.
3. Ввести `theme dark`, затем `theme light`, затем `theme toggle`.
4. Проверить `TopMenu`, `ProjectTreePanel`, `PropertiesPanel`, `Viewport3D`, `CommandConsole`.

## Проверка

```bash
cd frontend
npm run build
npm run lint
```
