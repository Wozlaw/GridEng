# Task 25.6 — Wind dialog: localization, 40 px fields and DTO modes

## Цель

Доработать диалог `Ветер`: русские пользовательские тексты, единые размеры полей, параметры DTO для типа местности, коэффициента надежности и режима расчета.

## Изменения

1. Заменить пользовательский текст `Assembly load case` на `Загружение`.
2. Убрать отсутствие отступа между верхним выпадающим списком и шапкой: использовать единую форму/Stack spacing.
3. Установить визуальную высоту всех controls формы ветра 40 px: select, number fields, inputs, icon buttons.
4. Давление в UI и DTO закрепить как `Па`. Не возвращать `kPa` в пользовательский интерфейс.
5. В DTO/состояние диалога добавить параметры:
   - `terrainType`: `A | B | C`;
   - `gammaF`;
   - `calculationMode`: `simple | sp20 | pue`.
6. В UI режимы отображать по-русски:
   - `Простой` → `simple`;
   - `СП 20.13330` → `sp20`;
   - `ПУЭ 7-е изд.` → `pue`.
7. Внутренние enum-ключи оставить латиницей.
8. Значения по умолчанию для коэффициента надежности:
   - `sp20`: `1.4`;
   - `pue`: `1.2`;
   - `simple`: текущее пользовательское значение или `1.0/1.4` только если это уже принято в модели. Не перемножать коэффициенты разных режимов.
9. При смене режима показывать один применяемый коэффициент надежности для выбранного режима.
10. Не менять 3D-вектор ветра в модели: `Z` по-прежнему не редактируется в UI и сохраняется как `0`, если это уже принято предыдущими задачами.
11. Проверить локализацию новых подписей при переключении языка, если в проекте включена i18n.

## Acceptance criteria

1. `Assembly load case` больше не отображается; пользователь видит `Загружение`.
2. Все поля формы высотой 40 px.
3. Давление отображается и сохраняется как Па.
4. Добавлены `terrainType`, `gammaF`, `calculationMode`.
5. Для ПУЭ по умолчанию применяется `1.2`, для СП — `1.4`.
6. Коэффициенты СП/ПУЭ не перемножаются.
7. Пройдены build/lint.
