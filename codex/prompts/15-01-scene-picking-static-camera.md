# Task 15.1 — Улучшить selection/picking на сцене без изменения расчетной геометрии

## Цель

Сделать выбор элементов удобным: визуально стержни могут быть толще/цилиндрическими, но расчетная геометрия не должна меняться. При выборе камера остается статичной.

## Сделать

1. Разделить расчетную геометрию и visual/picking layer:
   - расчетные node/member координаты не менять;
   - для выбора использовать утолщенные линии, `Line2`, cylinder proxy или invisible picking mesh.
2. В wireframe-режиме стержни сделать удобными для попадания мышью.
3. В real view в будущем использовать контур/геометрию профиля, не смешивать с wireframe proxy.
4. Выделение member:
   - больше толщина;
   - контрастная обводка/glow/halo;
   - render priority над невыбранными элементами;
   - не полагаться только на цвет.
5. Выделение node:
   - акцентировать точку;
   - если у node есть restraint, подсветить связанное закрепление.
6. Выбор load на сцене:
   - выбрать конкретный `load` через `selectLoad(loadCaseId, loadId)`;
   - подсветить вектор/точку приложения/стержень.
7. Запретить camera side effects:
   - selection не вызывает `fitToModel`;
   - selection не меняет `OrbitControls.target`;
   - selection не меняет zoom/rotation.

## Проверка вручную

- Выбрать стержень: камера не сдвинулась.
- Выбрать ноду с закреплением: подсвечиваются нода и закрепление.
- Выбрать нагрузку: подсвечивается нагрузка и target.
- `npm run build` и `npm run lint` проходят.
