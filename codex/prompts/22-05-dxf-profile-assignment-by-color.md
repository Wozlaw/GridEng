# Task 22.5 — DXF profile assignment by color

## Цель

Переработать диалог назначения профилей DXF-группам по принципу `цвет DXF → профиль каталога` с фильтром по типу профиля.

## Требования

1. Основной принцип: цвет задается в DXF для дифференциации профилей конструкции, поэтому mapping строится как `DXF color/color group → выбранный профиль`.
2. В строке каждой цветовой группы показать:
   - отображение цвета;
   - количество элементов группы, если доступно;
   - select “тип профиля”;
   - select “профиль”.
3. Тип профиля — это фильтр каталога, а не новое поле модели.
4. Профиль всегда назначается существующий из каталога.
5. При смене типа профиля:
   - список профилей фильтруется;
   - если текущий выбранный профиль не подходит новому типу, сбросить выбранный профиль.
6. Сейчас каталог может быть заглушкой, позже будет через API. Не закладывать жесткое ограничение только на три типа.
7. Типы `уголок / полоса / квадрат` можно показывать первыми, если они есть, но UI должен поддерживать больше типов из каталога/API.
8. Если каталог сейчас не содержит полноценного `profile.kind`, использовать существующее поле типа/категории профиля; при отсутствии — добавить локальную безопасную нормализацию в UI-слое без изменения модели.
9. Не менять API-контракт каталога профилей без отдельной задачи.

## Ограничения

- Не создавать новые профили из этого диалога.
- Не сохранять тип профиля в `GridEngModel` как отдельное новое поле.
- Не подключать реальный backend/API, если его нет в текущем проекте.

## Критерии готовности

- Диалог назначения профилей работает по цветовым группам DXF.
- Профили выбираются из существующего каталога.
- Фильтр по типу не скрывает будущие типы профилей.
- Неназначенные группы корректно подсвечиваются в diagnostics/logs.
- Build/lint проходят.

cd frontend
npm run build
