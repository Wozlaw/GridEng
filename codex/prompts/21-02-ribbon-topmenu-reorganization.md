# Task 21.2 — TopMenu/Ribbon: реорганизация групп и команд

## Цель

Перестроить `TopMenu` под согласованную структуру ribbon-групп, сохранив command registry, хоткеи и текущую модель `runAppCommand`.

## Общая модель группы

Рекомендуемая конфигурация группы:

```ts
type RibbonGroupConfig = {
  id: string;
  title: string;
  primary: CommandId[];
  inlineSecondary?: CommandId[];
  menuSecondary?: CommandId[];
};
```

Правила:

- в одном ряду больших кнопок не более 3;
- заголовок группы расположен под рядом кнопок;
- заголовок группы является кнопкой открытия `Menu/Popover` для `menuSecondary`;
- если большая кнопка одна, рядом с ней допускается столбец из 2 маленьких `inlineSecondary`;
- остальные малые команды — в выпадающем списке группы;
- `APP_COMMANDS_BY_ID`, `RIBBON_COMMAND_GROUPS`, хоткеи и `runAppCommand` не переписывать с нуля.

## Структура групп

### Логотип

- В начале меню показать компактный logo-only placeholder.
- Использовать MUI `FoundationIcon` или близкий доступный icon.
- Задать явный цвет из MUI palette.
- Убрать надписи `Sample tower segment` и `GridEng CAD Shell`.

### Компонент

Большие кнопки:

1. `Открыть` — существующая заглушка;
2. `Сохранить` — существующая заглушка;
3. `Импорт DXF`.

В выпадающем списке:

- `Экспорт DXF` — добавить заглушку через Alert;
- `Импорт JSON`;
- `Экспорт JSON`.

`Открыть` и `Сохранить` пока оставлять заглушками: не подменять их локальным JSON import/export.

### Режим просмотра

Большие кнопки в порядке:

1. `Каркас`;
2. `Реальный`;
3. `Карта напряжений`.

Из этого блока убрать `Fit`, `Нагрузки`, `Закрепления`.

В режимах `Реальный` и `Карта напряжений` не прорисовывать каркас и ноды. Отдельную подсветку выбранного стержня/узла в этой итерации не делать.

### Видимость

Большие кнопки:

1. `Вписать` — бывший `Fit`;
2. `Оси`;
3. `Сетка`.

В выпадающем списке:

- `Нагрузки`;
- `Закрепления`;
- `Надписи` — общий переключатель подписей, кроме подписей осей.

### Каталог

Большая кнопка:

- `Конструкции`.

Inline secondary, столбец 2 кнопки:

- `Профили` — профили текущего проекта;
- `Материалы` — материалы текущего проекта.

В выпадающем списке:

- `Каталог профилей` — общая база профилей, пока заглушка;
- `Каталог материалов` — общая база материалов, пока заглушка.

### Аналитика

Inline secondary:

- `Ветер` — редактирование ветра активного загружения.

### Отчет

- `Спецификация`.

Если реальной функциональности еще нет — показывать Alert-заглушку.

### О программе

Большая кнопка:

Inline secondary:

Существующие заглушки сохранить.

- Все команды вызываются через общий command registry.

- Меню не расширяет приложение за экран.
- Команды `Импорт/Экспорт JSON`, `Импорт DXF`, `Вписать`, режимы просмотра, видимость продолжают работать.
