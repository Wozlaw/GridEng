# Task 21.5 — Scene labels: подписи сил/моментов и новая визуальная грамматика закреплений

## Цель

Нормализовать подписи в сцене, добавить подписи сил и моментов, а также заменить/доработать отображение закреплений по осям через согласованные условные обозначения.

## Общие требования к подписям

1. Ввести единые scene-label константы:
   - базовый размер текста;
   - минимальный/максимальный экранный размер, если используется scaling;
   - общий стиль цвета/контраста.
2. Размер подписей сил, моментов, закреплений и осей должен выглядеть согласованно.
3. Подписи сил писать в соответствующей плоскости по рекомендации:
   - ориентировать подпись с учетом плоскости приложения силы;
   - не допускать полной потери читаемости при вращении камеры.
4. Все новые подписи сил/моментов/закреплений завязать на `visibility.labels`.
5. Подписи осей оставить завязанными на `visibility.axes`.

## Подписи сил и моментов

Добавить подписи для:

- сосредоточенных сил;
- моментов;
- распределенных нагрузок, если они уже визуализируются.

Формат подписи не должен перегружать сцену. Если в модели есть `name/comment`, использовать человекочитаемое имя. Если имени нет — использовать компонентное обозначение (`FX`, `MY`, `qZ` и т.п.) без лишних номиналов, если это конфликтует с деревом проекта.

## Закрепления по осям

Для каждой оси X/Y/Z анализировать пару степеней свободы:

- X: `ux`, `rx`;
- Y: `uy`, `ry`;
- Z: `uz`, `rz`.

Правила условных обозначений:

1. Перемещение вдоль оси запрещено и поворот вокруг этой оси запрещен:
   - `u=true`, `r=true`;
   - символ: куб вдоль соответствующей оси.
2. Перемещение вдоль оси запрещено, поворот вокруг этой оси разрешен:
   - `u=true`, `r=false`;
   - символ: шар вдоль соответствующей оси.
3. Поворот вокруг оси запрещен, перемещение вдоль оси разрешено:
   - `u=false`, `r=true`;
   - символ: цилиндр вдоль соответствующей оси.
4. Если оба свободны (`u=false`, `r=false`) — символ по этой оси не показывать.

Подписи рядом с символами:

- формат: `X:UX|RX`, `Y:UY|RY`, `Z:UZ|RZ`;
- если фиксирован только один DOF, показывать только его, например `X:UX` или `X:RX`;
- подпись располагается рядом с соответствующим символом.

## Цвета

Для элементов конструкции и тепловой карты использовать палитру из `@mui/material/colors`, если это не ломает существующую визуализацию.

## Ограничения

- Не реализовывать отдельную подсветку выбранного элемента в режимах `Реальный`/`Карта напряжений`.
- Не менять семантику DOF-модели.
- Не добавлять расчетную интерпретацию закреплений сверх визуализации.

## Ожидаемые места изменений

- `frontend/src/widgets/viewport-3d/*`
- `frontend/src/features/view-modes/*`
- компоненты `LoadVectors`, `MomentVectors`, `RestraintMarkers` или их фактические аналоги

## Критерии готовности

- Подписи сил/моментов видны при `visibility.labels=true` и скрываются при `false`.
- Закрепления различаются по X/Y/Z и по типу `cube/sphere/cylinder`.
- Формат подписи закреплений соответствует `X:UX|RX`.
- В режимах `Реальный` и `Карта напряжений` каркас и ноды не прорисовываются.
