# Task 25.7 — Project tree and PropertiesPanel polish with multiselect

## Цель

Доработать панели `Состав проекта` и `Свойства`: единая высота элементов 40 px, разделители раскрытых секций, AutoCAD-like таблица свойств и множественный выбор через Shift.

## Часть A. Панель `Состав проекта`

1. После раскрытых секций `Нагрузки` и `Элементы` рисовать разделительную линию на всю ширину панели.
2. Разделитель показывать после содержимого раскрытой секции, а не после заголовка.
3. Высота строк дерева/списка, кнопок действий и элементов секций — 40 px.
4. Для длинных названий использовать обрезку с tooltip, без увеличения высоты строки.

## Часть B. Панель `Свойства`

1. В заголовке убрать вторую строку. Оставить только основную строку выбранного объекта/состояния.
2. Вторичную информацию перенести в таблицу свойств или tooltip, если она нужна.
3. Выполнить свойства в виде AutoCAD-like таблицы:
   - левая колонка — имя свойства;
   - правая колонка — значение/редактор;
   - компактные заголовки групп допустимы;
   - строки и controls высотой 40 px.
4. Сохранить существующие механизмы редактирования, валидации и обновления модели.
5. Проверить минимум сущностей: узел, стержень, загружение, сила/момент, закрепление/DOF, профиль/материал, отсутствие выбора.

## Часть C. Множественный выбор

1. Добавить возможность выбора нескольких элементов на сцене через `Shift`.
2. Множественный выбор должен работать поверх существующей selection-логики, не ломая одиночный выбор.
3. В панели свойств при множественном выборе показывать общие свойства выбранных элементов.
4. Если значение свойства различается между выбранными элементами, в поле значения показывать строку `******РАЗЛИЧНЫЕ******`.
5. При изменении значения свойства в поле с множественным выбором применить изменение ко всем выделенным элементам, для которых это свойство допустимо.
6. Не применять изменение к сущностям, для которых поле неприменимо; показать compact warning через общий MUI Alert/Snackbar.
7. Не менять онтологию модели ради multiselect; использовать существующие store actions или добавить минимальные batch-actions поверх них.

## Acceptance criteria

1. ProjectTree имеет строки 40 px и разделители после раскрытых `Нагрузки`/`Элементы`.
2. PropertiesPanel имеет однострочный заголовок и AutoCAD-like таблицу.
3. Редакторы существующих сущностей не потеряны.
4. Shift-click добавляет/убирает элементы из выбора.
5. Различающиеся значения отображаются как `******РАЗЛИЧНЫЕ******`.
6. Изменение общего свойства применяется ко всем выбранным допустимым элементам.
7. Пройдены build/lint.
