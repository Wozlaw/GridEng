# Task 14.1 — Перестроить ProjectTree

## Цель

Сделать дерево проекта пользовательским, без дублей и служебных raw-полей.

## Сделать

1. Заголовок панели: `Состав проекта`, по центру, снизу разделительная линия.
2. Убрать блок `Структура модели`.
3. Убрать из дерева блоки `Профили` и `Материалы`. Они открываются через отдельные диалоги/каталоги.
4. Оставить сворачиваемые блоки:
   - `Нагрузки`;
   - `Элементы`;
   - `Ноды`.
5. Нагрузки:
   - показывать структуру `Расчетный случай → нагрузки`;
   - при выборе load case выбирать `loadCase`;
   - при выборе конкретной нагрузки выбирать `load` с `loadCaseId/loadId`;
   - строка нагрузки: `name`, тип (`сила`, `момент`, `распределенная`) и короткая величина;
   - подсветка selected load.
6. Элементы:
   - строка: `{label элемента или M-номер} — {profile.name}`;
   - перед названием цветной marker профиля;
   - tooltip: `startNode.label → endNode.label`, id допускается только в tooltip.
7. Ноды:
   - показывать только `label`;
   - fallback: `Нода {порядковый номер}`;
   - raw id только в tooltip.
8. При клике по нагрузке/элементу/ноде обновлять selection и Properties.

## Ограничения

- Не показывать материалы/профили в дереве.
- Не выводить raw ids в основном тексте.

## Проверка

- Дерево не дублирует status bar.
- Выбор load из дерева открывает свойства конкретной нагрузки.
- `npm run build` и `npm run lint` проходят.
