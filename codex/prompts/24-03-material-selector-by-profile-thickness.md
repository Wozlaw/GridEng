# Task 24.3 — Material selector filtered by profile thickness

## Цель

Реализовать выбор материала элемента после выбора профиля с фильтрацией сталей по расчетной толщине профиля и типу проката.

## Сделать

1. Найти текущий редактор `Member`/Properties, где задаются `profileId` и `materialId`.
2. После выбора профиля определить расчетную толщину профиля:
   - использовать данные API/details профиля;
   - если у профиля несколько толщин, явно оформить helper `resolveMaterialThicknessMm(profile)` с комментариями по `profile_type`.
3. Определить `product_type` для запроса материалов:
   - по данным профиля, если backend их возвращает;
   - иначе через централизованный mapping `profile_type -> product_type`, без hardcode в компоненте.
4. Через `materialsApi` получить допустимые стали по `thickness_mm` и `product_type`.
5. В селекторе материалов показывать `display_name`, расчетные параметры `Rt/Rb`, источник/ГОСТ в tooltip/details.
6. Сохранять выбранный материал в `Member`, не в профиль.
7. Если текущий материал стал недопустим после смены профиля, показать warning и предложить выбрать новый материал.

## Ограничения

- Не вводить `defaultMaterialId` у профиля в этой задаче.
- Не добавлять химию/категории поставки/ударную вязкость в UI.
- Не менять расчетное ядро.

## Критерии готовности

- Материалы фильтруются после выбора профиля.
- Материал остается свойством элемента.
- Некорректное сочетание профиль/материал не проходит тихо.
- Build/lint проходят.
