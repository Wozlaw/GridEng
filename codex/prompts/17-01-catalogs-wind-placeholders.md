# Task 17.1 — Диалоги профилей/материалов, ручной ветер и placeholder-команды

## Цель

Реализовать пользовательские диалоги для проектных каталогов и ручного ввода ветра. Будущие функции показывать через MUI Alert, не через window.alert.

## Диалог профилей проекта

Открывается из ribbon `Каталог → Профили`.

Показать:

- `name`;
- `kind`;
- параметры сечения;
- `areaMm2`;
- `Jx/Iy/Iz`;
- `Wx/Wy/Wz`;
- `massKgPerM`;
- default rotation/offset;
- цвет профиля;
- количество элементов с этим профилем;
- суммарная длина;
- суммарная масса по профилю;
- `comment`.

## Диалог материалов проекта

Открывается из ribbon `Каталог → Материалы`.

Показать:

- `name`;
- `elasticModulusMPa`;
- `shearModulusMPa`;
- `poissonRatio`;
- `densityKgPerM3`;
- `yieldStrengthMPa`;
- количество элементов с материалом;
- суммарная длина;
- суммарная масса элементов;
- `comment`.

## Ручной ветер

Открывается из ribbon `Аналитика → Ветер`.

Для выбранного load case редактировать:

- direction X/Y/Z;
- nominalPressureKPa;
- comment.

Требования:

- направление нормализовать;
- нулевой direction означает ветер отключен;
- нормативный калькулятор не реализовывать;
- показывать единицы `kPa`.

## Placeholder-команды

Показывать MUI Alert info:

- `Каталог конструкций`: будущий backend-атлас типовых опор с типовыми нагрузками;
- `Отчет по расчету`: будет доступен после расчетных результатов/backend;
- `Спецификация`: если пока не реализована — placeholder, если можно сделать из текущей модели — допускается простая таблица как отдельная будущая задача;
- `Справка/Лицензия/О программе`: placeholder, если контент отсутствует.

## Проверка

- Диалог профилей и материалов открывается и не ломает модель.
- Ветер задается вручную в выбранном load case.
- Placeholder показывает Alert снизу по центру.
- `npm run build` и `npm run lint` проходят.
