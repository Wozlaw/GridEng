# Task 21.6 — ProjectTree и PropertiesPanel: типографика, scroll и очистка шапки

## Цель

Довести боковые панели до согласованного интерфейсного состояния: уменьшить «кричащие» заголовки, добавить внутренний scroll и убрать служебную информацию из шапки Properties.

## ProjectTreePanel / Состав проекта

1. Уменьшить размер заголовка панели.
2. Добавить вертикальную прокрутку внутри панели.
3. Scroll должен быть у содержимого панели, а не у всего приложения.
4. Раздел `Нагрузки` должен использовать active load case из задачи 21.3.
5. В списке нагрузок показывать только наименования усилий/нагрузок без номиналов.

## PropertiesPanel / Меню свойств

1. Уменьшить размер заголовка панели.
2. Из-под заголовка убрать всю служебную информацию.
3. В шапке оставить только человекочитаемое наименование выбранного объекта.
4. Служебные данные не удалять из модели и не уничтожать в редакторах: при необходимости оставить их в свернутом техническом блоке или в местах, где они нужны для отладки.
5. Добавить вертикальную прокрутку внутри `PropertiesPanel`.
6. Если не усложняет верстку, шапку можно сделать sticky.

## Ограничения

- Не упрощать редакторы ценой потери данных.
- Не удалять id/comment/source fields из модели.
- Не переносить scroll на body/root.

## Ожидаемые места изменений

- `frontend/src/widgets/project-tree/*`
- `frontend/src/widgets/properties-panel/*`
- shared typography/spacing tokens, если есть

## Критерии готовности

- Заголовки обеих панелей визуально спокойнее и согласованы.
- Обе панели прокручиваются внутри своей области.
- Шапка Properties показывает только имя/наименование.
- Служебные поля не потеряны из модели и экспорта.
