# Task 21.4 — Scene overlays: ветер, легенда карты напряжений и visibility.labels

## Цель

Согласовать overlay-элементы сцены и добавить общий переключатель подписей `visibility.labels`.

## Что сделать

### `visibility.labels`

1. Добавить в UI-состояние видимости флаг `labels`.
2. Подключить команду `Надписи` из группы `Видимость`.
3. `labels=false` скрывает подписи:
   - сил;
   - моментов;
   - распределенных нагрузок;
   - закреплений.
4. Подписи осей не зависят от `visibility.labels`; они зависят только от `visibility.axes`.

### Ветер

1. В правом верхнем углу viewport показать вектор направления ветра, если у активного загружения есть:
   - ненулевой direction-вектор;
   - ненулевое/заданное nominal pressure.
2. Источник данных — `activeLoadCase.wind`, не первое загружение и не глобальная модель.
3. Цвет вектора: `#ffffff`.
4. Реализовать как HTML overlay поверх Canvas, чтобы объект не зависел от камеры.
5. Показать краткую подпись давления в `Pa`, если она уже есть в модели.

### Легенда карты напряжений

1. Закрепить легенду карты напряжений за левым нижним углом viewport.
2. Реализовать как HTML overlay поверх Canvas.
3. Использовать палитру из `@mui/material/colors` для элементов конструкции/тепловой карты, где это не ломает текущий stress-map.

### Gizmo

- Оставить gizmo в правом нижнем углу.
- Не бороться с тем, что он туда «слетает»; это согласованное положение.

## Ограничения

- Не реализовывать расчет ветра.
- Не менять формат результатов расчета.
- Не привязывать wind overlay к 3D-координатам.

## Ожидаемые места изменений

- `frontend/src/app/store/*`
- `frontend/src/features/view-modes/*`
- `frontend/src/widgets/viewport-3d/*`
- компоненты legend/wind overlay, если уже есть

## Критерии готовности

- Переключатель `Надписи` работает и не отключает подписи осей.
- Wind overlay показывает ветер только активного загружения.
- Stress legend закреплена слева снизу.
- Gizmo остается справа снизу.
