# Task 22.4 — DXF 3D preview and color modes

## Цель

Заменить список предупреждений в основном DXF-диалоге на 3D-сцену preview с переключением режимов окраски: `Диагностика` и `Профили`.

## Требования к 3D preview

1. Preview строится до применения импорта к основной модели.
2. Preview не должен менять `modelStore` до нажатия “Импорт”.
3. По возможности использовать существующий renderer/viewport-компоненты, но на временной preview-модели или отдельном preview state.
4. Основной DXF-диалог должен показывать:
   - имя файла;
   - 3D preview;
   - переключатель режима окраски;
   - compact-status;
   - нижний блок icon buttons.
5. Список warnings/errors не показывать в основном диалоге; подробности — в `ImportLogDialog`/`ImportLogDrawer`.

## Режим `Диагностика`

Окрашивать элементы/группы по structured diagnostics:

- зеленый — `ok`;
- желтый — `warning`;
- красный — `error`.

Если конкретная диагностика не привязана к элементу, показывать ее в логах, а не окрашивать всю модель целиком.

## Режим `Профили`

1. Цвет элемента брать из исходного DXF color, потому что цвет в DXF используется для дифференциации профилей конструкции.
2. Каждый DXF color/color group должен соответствовать назначению профиля в следующей задаче.
3. Если для цвета профиль еще не назначен, элемент остается окрашенным по DXF color, а статус группы в логах/compact-status должен показывать “профиль не назначен”.

## UI-переключатель

- Названия режимов: `Диагностика` и `Профили`.
- Переключатель сделать компактным и совместимым с grayscale UI.
- Не засорять TopMenu новыми командами; это локальный режим просмотра DXF-диалога.

## Ограничения

- Не реализовывать назначение профилей в этой задаче, кроме чтения уже существующих mapping-данных.
- Не менять формат сохраняемой модели.
- Не добавлять внешние зависимости без необходимости.

## Критерии готовности

- В основном DXF-диалоге есть 3D preview вместо списка предупреждений.
- Режим `Диагностика` использует per-element/per-group diagnostics.
- Режим `Профили` использует исходные DXF colors.
- Логи открываются отдельно.
- Build/lint проходят.
