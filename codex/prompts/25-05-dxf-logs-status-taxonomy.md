# Task 25.5 — DXF logs: compact text and status taxonomy

## Цель

Сделать вкладку `Логи` компактной и понятной, убрать дублирующую кнопку логов, согласовать статусы импорта так, чтобы исправная LINE-модель не окрашивалась в желтый из-за служебных сообщений.

## Изменения

1. Убрать дублирующую кнопку `Логи импорта` из нижней панели, так как логи доступны через вкладку `Логи`.
2. Убрать чипы из текста логов. Вместо них использовать обычный текст: `Статус: ошибка`, `Статус: предупреждение`, `Статус: информация`.
3. Не выводить raw JSON.
4. Сообщения по одному DXF-элементу/handle агрегировать в одну запись.
5. Код `INE` и аналогичные коды расшифровывать пользовательским текстом. Например: `Статус: информация. Элемент DXF проигнорирован: тип объекта не поддерживается импортом LINE-only.`
6. Если объект является валидной `LINE` и участвует в модели, предупреждение `ignored entity` для него не выводить.
7. Если объект не `LINE` и корректно проигнорирован в режиме LINE-only, считать это `info`, а не `warning`, если все LINE импортированы корректно.
8. Ввести/зафиксировать таксономию статусов:

### error — импорт блокируется

- нулевая длина LINE;
- не назначен профиль;
- не назначен материал;
- невалидные координаты;
- невозможность построить узлы/стержни;
- стержни без нод/связей;
- критическая ошибка чтения DXF.

### warning — импорт разрешен, но есть инженерно значимое отклонение

- висячие стержни;
- несколько несвязанных компонент;
- Z-диапазон был нормализован/исправлен;
- геометрия требует нормализации/поворота для ожидаемой ориентации.

### info — не влияет на цвет модели

- количество LINE;
- слои;
- цвета;
- DXF handle;
- параметры импорта;
- unsupported entity ignored при корректном импорте всех LINE.

9. Цвет всей модели/элементов в режиме диагностики должен определяться только `error/warning` по этой таксономии. `info` не должен делать модель желтой.
10. Исправная модель без `error/warning` должна быть зеленой.

## Acceptance criteria

1. Логи компактные, без чипов и без JSON.
2. Один DXF handle не порождает пачку повторяющихся сообщений.
3. `unsupported entity ignored` при LINE-only импорте не желтит корректную модель.
4. Исправная модель отображается зеленой в диагностическом режиме.
5. Ошибки блокируют импорт; предупреждения не блокируют.
6. Пройдены build/lint.
