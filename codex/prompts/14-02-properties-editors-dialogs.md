# Task 14.2 — Доработать Properties panel и базовые диалоги

## Цель

Properties должны редактировать выбранную пользовательскую сущность: node, member, restraint, load. Профили и материалы полностью показываются в отдельных диалогах, но в свойствах элемента остаются краткие карточки назначения.

## Сделать

### Общие требования

- Заголовок панели по центру, снизу разделительная линия.
- Все формы на русском/i18n.
- Для выбранной сущности показывать `Комментарий`, если сущность поддерживает `comment`.
- Не показывать raw debug-поля в основной панели.

### Node

Редактируемые поля:

- `label` / Наименование;
- позиция X/Y/Z ГСК;
- `comment`;
- закрепления через 6 toggle-кнопок `UX/UY/UZ/RX/RY/RZ`.

Пресеты закреплений:

- `Свободно` — удалить/обнулить restraint;
- `Шарнир` — UX/UY/UZ true, RX/RY/RZ false;
- `Заделка` — все true;
- `Пользовательское` — текущие toggles.

Важно: редактировать `model.restraints[]` через upsert/delete, не добавлять поле в Node.

### Member

Поля:

- кратко назначенный профиль + кнопки `Выбрать/заменить`, `Открыть карточку`;
- кратко назначенный материал + кнопки `Выбрать/заменить`, `Открыть карточку`;
- `localAxisRotationDeg` — поворот вокруг X;
- `offsetYmm`, `offsetZmm`;
- `comment`.

`DXF Source` убрать из основной панели; доступ через консоль/debug-команду в следующих задачах.

### Load

Поддержать редактор:

- `name`;
- `comment`;
- `type`: узловая сосредоточенная / распределенная на стержень;
- `kind`: сила / момент;
- `coordinateSystem`: только ГСК, readonly;
- direction X/Y/Z с нормализацией;
- для nodal concentrated: `target node`, `magnitude`;
- для member distributed: `target member`, `qStart`, `qEnd`, `xStartRel`, `xEndRel`.

Для `function` distribution показывать readonly placeholder Alert: функциональная нагрузка зарезервирована, редактор будет позже.

### Диалоги

- Диалог выбора профиля для элемента.
- Диалог выбора материала для элемента.
- Диалог карточки профиля.
- Диалог карточки материала.

Если выбирается сущность из будущего глобального каталога, сейчас показывать placeholder; для текущей задачи выбирать только из `model.profiles/model.materials`.

## Проверка

- Редактирование node/member/load/restraint меняет модель через store actions.
- Ошибки ввода показываются MUI Alert.
- Камера не меняется при редактировании/выборе.
- `npm run build` и `npm run lint` проходят.
