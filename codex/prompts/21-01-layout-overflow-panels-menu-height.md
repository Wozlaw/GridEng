# Task 21.1 — Layout: overflow, ширина приложения и фиксированная высота TopMenu

## Цель

Устранить ситуацию, когда область `Properties` или все приложение уезжает за правый край экрана из-за увеличенного ribbon-меню. За край экрана не должна уезжать ни одна основная панель.

## Что сделать

1. Проверить текущий layout `CadShell` и связанные CSS/MUI `Box/Stack/Grid`-контейнеры.
2. Центральную область приложения сделать устойчивой к переполнению:
   - центральная колонка сцены: `minmax(0, 1fr)` или эквивалент;
   - боковые панели: фиксированные/ограниченные `min/max width`;
   - вложенные контейнеры: `minWidth: 0`, где это необходимо;
   - глобальный горизонтальный overflow приложения не должен появляться.
3. Для `TopMenu` ввести фиксированную CSS-константу высоты, например `--top-menu-height`, и использовать ее в расчетах высоты shell/viewport.
4. Ribbon может иметь внутренний горизонтальный overflow или адаптивное сворачивание, но не должен расширять корневой layout.
5. Проверить ширину 1366 px: левая панель, сцена, правая панель и нижняя зона должны оставаться в пределах viewport.

## Ограничения

- Не менять семантику команд меню.
- Не переносить боковые панели внутрь Canvas.
- Не скрывать `PropertiesPanel` как способ решения overflow.

## Ожидаемые места изменений

- `frontend/src/app/layout/*`
- `frontend/src/widgets/*`
- `frontend/src/app/CadShell*` или фактический файл shell
- CSS/MUI style-файлы, если они уже выделены

## Критерии готовности

- При раскрытых левой и правой панелях приложение не дает горизонтальный scroll на уровне body/root.
- `PropertiesPanel` не выезжает за экран.
- `TopMenu` не увеличивает общую ширину приложения.
- Сцена корректно занимает оставшуюся центральную область.

## Проверка

```bash
cd frontend
npm run build
npm run lint
```

Ручная проверка: открыть приложение на ширине 1366 px, раскрыть/свернуть боковые панели, открыть меню и убедиться, что правый край `Properties` виден.
