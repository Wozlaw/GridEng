# Task 23.3 — DXF profiles tab color-profile table

## Цель

Оформить вкладку `Профили` как чистую таблицу соответствия DXF color → profile assignment.

## Требования

1. На вкладке `Профили` показать только таблицу назначений профилей, без счетчиков и служебных summary.
2. Структура таблицы строго:
   - `Цвет`;
   - `Тип профиля`;
   - `Стандарт`;
   - `Профиль`.
3. `Цвет`:
   - показывать только цветовой индикатор/swatch;
   - текстовые коды цвета, RGB/hex, layer id и counts не показывать в основной таблице;
   - при клике на индикатор открывать цветовую палитру/пикер для изменения цвета группы.
4. Изменение цвета должно обновлять mapping preview-группы и режим `Профили`, но не должно ломать исходный DXF color в логах.
5. `Тип профиля`:
   - select-фильтр каталога типов;
   - не ограничиваться только `уголок / полоса / квадрат`, потому что каталог будет расширяться через API;
   - если текущий профиль не соответствует новому типу, сбрасывать поле `Профиль`.
6. `Стандарт`:
   - select или read-only поле в зависимости от текущего каталога;
   - должен фильтровать список профилей, если в каталоге есть стандарт/ГОСТ.
7. `Профиль`:
   - select существующего профиля из каталога/заглушки;
   - не создавать новое поле модели и не вводить произвольный профиль строкой.
8. Все изменения в таблице должны отражаться в preview без применения импорта к основной модели.
9. Ошибки неполного назначения показывать валидацией строки и в логах, а не общими JSON-блоками.

## Ограничения

- Не реализовывать настоящий backend API каталога, если его еще нет; использовать существующую заглушку/локальный каталог.
- Не менять онтологию профиля: тип/стандарт — фильтры выбора, профиль назначается существующий.
- Не выводить counts/служебные поля на вкладке `Профили`.

## Критерии готовности

- Вкладка `Профили` содержит таблицу `Цвет | Тип профиля | Стандарт | Профиль`.
- Цветовой индикатор кликабелен и открывает палитру.
- Фильтрация профилей по типу/стандарту работает.
- Служебные сведения перенесены в `Логи`.
- Build/lint проходят.
