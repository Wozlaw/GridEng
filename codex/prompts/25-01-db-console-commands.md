# Task 25.1 — DB console commands via command registry

## Цель

Добавить консольные команды базы данных в соответствии с текущей frontend/backend-реализацией, не вводя новых backend-контрактов и не обходя существующий `command registry`.

## Контекст

Работать поверх закрытых блоков `21–24`. Использовать существующие API/adapters для `cross_sections` и `materials`, реализованные в блоке 24.

Опорные документы:

- `codex/reference/UI_REVISION_4_CHECKED_REQUIREMENTS.md`;
- `codex/docs/catalogs-api-integration-spec.md`;
- `codex/UNIFIED_GRIDENG_SPECIFICATION.md`.

## Изменения

1. Найти текущий `command registry` / `APP_COMMANDS_BY_ID` / `runAppCommand` и добавить DB/catalog команды только через него.
2. Не создавать локальные обработчики команд внутри UI-компонентов.
3. Добавить только безопасные команды, которые поддержаны текущими сервисами/API:
   - `db.status` — краткая проверка доступности каталогов/источников данных;
   - `db.profiles` — краткий summary доступных профилей/типов/стандартов;
   - `db.materials` — краткий summary доступных материалов;
   - `db.refresh` или `db.reload` — только если уже существует сервис безопасного refresh/reload справочников.
4. Если backend endpoint или frontend service для команды отсутствует — не реализовывать фиктивное изменение БД. Команда должна вернуть понятное сообщение `Команда недоступна: отсутствует API ...`.
5. Все результаты выводить в консоль/уведомления компактно, без raw JSON.
6. Добавить команды в `help` консоли.
7. Сохранить поддержку локализации ru/en, если в проекте уже есть языковые словари.

## Запрещено

- Добавлять команды, напрямую изменяющие БД, без реализованного backend API и сценария подтверждения.
- Подменять API mock-данными как будто команда успешно работает с БД.
- Менять онтологию каталогов профилей и материалов.

## Acceptance criteria

1. Команды доступны через консоль и отображаются в `help`.
2. Команды используют существующий command registry.
3. Отсутствующий backend/service не приводит к падению UI.
4. Raw JSON не выводится пользователю.
5. Пройдены:

```bash
cd frontend && npm run build
cd frontend && npm run lint
```
