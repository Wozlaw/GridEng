# Task 22.2 — DXF settings via console and import log dialog

## Цель

Убрать настройки DXF-импорта из основного диалога, перенести управление настройками в консольные команды, а служебную информацию и предупреждения — в отдельное окно логирования.

## Требования к настройкам DXF

1. Удалить редактируемые настройки импорта из пользовательского DXF-диалога.
2. Существующие `DxfImportSettings` не менять по смыслу:
   - `toleranceMm`;
   - `centerOnXY`;
   - `force2DToXY`.
3. Добавить команды через существующий `command registry`:
   - `dxf.settings` — показать текущие настройки;
   - `dxf.tolerance <number>` — изменить tolerance в мм;
   - `dxf.center on|off` — изменить `centerOnXY`;
   - `dxf.force2d on|off` — изменить `force2DToXY`;
   - `dxf.settings reset` — вернуть настройки по умолчанию.
4. Команды должны быть отражены в `help`.
5. В основном DXF-диалоге не показывать редактируемый блок настроек. Текущие настройки показывать только в логах import summary.

## Требования к окну логирования

1. Добавить `ImportLogDialog` или `ImportLogDrawer` как переиспользуемый паттерн для будущих импортов, но подключить сейчас к DXF.
2. Открывать окно по icon button `Логи` из нижнего блока кнопок DXF-диалога.
3. В логах показывать:
   - имя файла;
   - summary импорта;
   - текущие параметры импорта;
   - warnings;
   - errors;
   - ignored entities count;
   - color/layer groups;
   - source metadata, если уже есть в preview.
4. Служебные метрики из основного диалога перенести в логи:
   - `linesCount`;
   - `nodesCount`;
   - `membersCount`;
   - `ignoredEntitiesCount`;
   - `zRange`;
   - `mergedNodesCount`;
   - `danglingMembersCount`.
5. Основной диалог DXF должен оставлять только:
   - имя файла;
   - preview-зону;
   - переключатель режима окраски, если он уже появится в следующей задаче;
   - compact-status;
   - нижний блок кнопок.

## Нижний блок кнопок DXF-диалога

Сформировать единый нижний блок icon buttons:

- выбрать/заменить файл;
- открыть логи;
- отмена;
- импорт.

Для выбора файла использовать скрытый `input type="file"`, видимую кнопку сделать через MUI `IconButton`/`ButtonBase` с `Tooltip` и `aria-label`.

## Ограничения

- Не реализовывать 3D preview в этой задаче, если его еще нет: подготовить структуру диалога и логов.
- Не менять формат `GridEngModel`.
- Не менять DXF-парсер, кроме минимально необходимой передачи данных в логи.

## Критерии готовности

- Настройки DXF меняются только через консольные команды.
- В основном DXF-диалоге нет редактируемых настроек и длинного списка warnings/errors.
- Подробности доступны через окно логов.
- Все icon buttons имеют `Tooltip` и `aria-label`.
- Build/lint проходят.
